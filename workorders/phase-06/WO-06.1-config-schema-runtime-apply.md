## Implementation Checklist
1. Define typed config schema (JSON Schema or typed validator).
2. Promote legacy keys into canonical structure on load.
3. Apply validated config into runtime atomically.
4. Persist normalized config back to disk when migrated.

## Success Metrics
- Router starts with old config and self-heals to canonical format.
- Invalid config fields produce clear validation errors.
- Runtime uses normalized values without manual edits.

## Potential Failure Modes And Preventions
- Failure mode: silent coercion hides config errors.
  Prevention: warning log for every promoted/invalid key.
- Failure mode: partial apply yields inconsistent runtime.
  Prevention: validate full config before mutate runtime state.
- Failure mode: schema changes break user-edited files.
  Prevention: versioned schema and migration map.

## Verification
- Test load with intentionally legacy config keys.
- Confirm saved file is normalized and router behavior unchanged.
