## Success Metrics
- Endpoint list returns NKN + cloudflare/local candidates and key metadata in one contract.
- Unauthorized clients cannot update another session's endpoint record.
- Shared-key verification failures are explicit and observable in logs/metrics.

## Potential Failure Modes And Preventions
- Failure mode: key metadata leaks into public telemetry.
  Prevention: redact signatures/secrets from public endpoints; expose only non-sensitive fingerprints.
- Failure mode: endpoint spoofing via unauthenticated payloads.
  Prevention: enforce `requireAuth` and bind writes to authenticated `sessionId/userId/deviceId`.
- Failure mode: stale cloudflare URLs remain advertised.
  Prevention: heartbeat expiry + `last_verified_at` TTL pruning.
- Failure mode: schema drift breaks existing clients.
  Prevention: additive migration + backward-compatible response fields + contract version key.
