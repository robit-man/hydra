## Implementation Checklist
1. Define canonical Hydra bridge event types for multimodal asset updates (resource, command, media, geometry).
2. Add payload metadata required for idempotency and replay control: `messageId`, `chunk`, `total`, `assetId`, `checksum`, `contentType`.
3. Add chunking/reassembly for oversized pointcloud + GLB payloads with per-chunk ACK and timeout retry.
4. Propagate selected transport state (`nkn`, `cloudflare`, `local`) in every media/asset payload.
5. Preserve current message types with compatibility mapping during migration.

## Success Metrics
- Hydra can dispatch text, commands, audio, video, pointcloud chunks, and GLB updates over one bridge path.
- Out-of-order chunk arrival still reconstructs correct pointcloud/GLB payloads.
- Media/control packets include stable IDs and are de-duplicated when retransmitted.
- NoClip consumes payloads without requiring transport-specific logic in UI components.

## Potential Failure Modes And Preventions
- Failure mode: payloads exceed DM limits and fail silently.
  Prevention: hard size guards + mandatory chunked transfer above threshold + explicit error events.
- Failure mode: duplicate packets mutate overlay state multiple times.
  Prevention: message-id cache with TTL and idempotent apply semantics.
- Failure mode: stale cloudflare endpoint breaks video frame fetch.
  Prevention: use stale-tunnel classifier and transport fallback metadata before retries.
- Failure mode: incompatible legacy event names split traffic.
  Prevention: compatibility translator in `noclipBridge.js` for old and new event aliases.
