4. Emit pass/fail artifact JSON with per-stage timings and error details.

## Success Metrics
- One command executes the complete interop smoke test and reports stage-level results.
- Failure drills produce expected degraded behavior (no hard crash, clear fallback path).
- Health and endpoint contract checks are required gates before rollout.
- Rollback procedure can disable interop changes without data corruption.

## Potential Failure Modes And Preventions
- Failure mode: green smoke tests miss transport-specific regressions.
  Prevention: run scenarios across `nkn`, `cloudflare`, and fallback local/relay modes.
- Failure mode: backend and frontend versions drift and tests pass only partially.
  Prevention: pin contract version in test fixtures and fail on mismatch.
- Failure mode: retries hide latent latency spikes and user-visible failure.
  Prevention: enforce latency/error SLO thresholds in smoke output.
- Failure mode: rollback leaves stale endpoint records/sessions.
  Prevention: include cleanup steps and post-rollback validation checks in runbook.

## Verification
- Run smoke harness and confirm all required stages pass with artifacts generated.
- Run each failure drill independently and verify expected degraded state + recovery.
- Execute rollback steps in a staging environment and confirm clean recovery.
