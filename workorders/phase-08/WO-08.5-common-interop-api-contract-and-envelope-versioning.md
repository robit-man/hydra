## Implementation Checklist
1. Add a contract version (`interopContractVersion`) and envelope schema (`event`, `source`, `target`, `transport`, `payload`, `ts`).
2. Define canonical endpoint-state fields for both stacks: `selected_transport`, `candidates`, `health`, `stale_tunnel_url`, `nkn_address`.
3. Enforce strict validation for required fields and emit stable machine-readable errors on invalid payloads.
4. Add backward-compatible decode for current payloads during migration (`legacy=true` path).
5. Publish schema docs and JSON examples for every interop message type.

## Success Metrics
- Hydra `/services/snapshot` and NoClip endpoint list can be normalized into the same contract without lossy mapping.
- A single parser in each frontend can decode all supported interop events.
- Invalid payloads are rejected with deterministic error codes and do not crash handlers.
- Contract docs are sufficient for implementing a new compatible client without reading source code.

## Potential Failure Modes And Preventions
- Failure mode: schema drift between router and backend causes silent parse failures.
  Prevention: `interopContractVersion` required on all envelopes plus contract tests in CI.
- Failure mode: sensitive fields leak in public payloads.
  Prevention: keep redaction in `router.py` public payload path and expose only key fingerprints, never secrets.
- Failure mode: partial migration breaks existing clients.
  Prevention: keep dual-decoder compatibility for one migration window and log legacy usage.
- Failure mode: endpoint transport semantics differ across repos.
  Prevention: canonical enum set (`cloudflare|upnp|nats|nkn|local`) with rejection of unknown values.
