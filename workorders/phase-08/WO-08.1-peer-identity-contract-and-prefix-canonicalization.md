## Implementation Checklist
1. Write a canonical identity schema covering: `network`, `nknPub`, `addr`, `displayName`, `userId`, `deviceId`, `source`, `observedAt`.
2. Standardize prefix behavior:
- Hydra peers must serialize as `hydra.<hex64>`.
- NoClip peers must serialize as `noclip.<hex64>`.
- Bare 64-byte hex must be accepted as input but normalized on first write.
3. Add deterministic normalization utilities in both codebases and replace ad hoc parsing.
4. Enforce self-peer suppression via canonical normalized key (not raw incoming address).
5. Add migration path for existing localStorage/session records to canonical format.

## Success Metrics
- Same peer observed via NKN DM, NATS discovery, or URL ingest resolves to one stable canonical key.
- No duplicate entries are shown for the same peer under prefix/no-prefix variants.
- Self identity is never shown as a discoverable remote peer.
- Existing persisted peer/session data auto-migrates without manual user edits.

## Potential Failure Modes And Preventions
- Failure mode: prefix collisions (`graph.`/`hydra.`/`noclip.`) create duplicate records.
  Prevention: canonicalization must run before all map keys and persistence writes.
- Failure mode: case-sensitive key drift (`ABC...` vs `abc...`).
  Prevention: lowercase normalization for all hex and prefixed addresses.
- Failure mode: mixed transport payloads overwrite trusted identity fields.
  Prevention: apply field precedence rules (`authenticated transport > discovery > URL hint`).
- Failure mode: old sessions break after schema change.
  Prevention: versioned storage key + one-way migration function with fallback rollback copy.
