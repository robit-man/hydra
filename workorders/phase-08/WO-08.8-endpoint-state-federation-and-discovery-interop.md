## Implementation Checklist
1. Fix NATS subject mismatch by moving DM subjects to `discovery.<room>.dm.<pub>` in both apps.
2. Ensure each discovered peer carries `network` metadata and canonical prefixed address.
3. Include cloudflare/nkn/local endpoint candidates in federated endpoint records.
4. Keep manual peer injection as a fallback when discovery channels are unavailable.
5. Add telemetry for transport chosen, discovery source, and stale endpoint rejection.

## Success Metrics
- Hydra and NoClip peers can discover each other within the same room using one subject convention.
- NoClip endpoint directory can advertise and consume Hydra cloudflared + NKN candidates.
- Transport selection is deterministic and visible to users/telemetry.
- Discovery degradation (NATS outage) falls back to manual + direct NKN paths.

## Potential Failure Modes And Preventions
- Failure mode: room mismatch or subject mismatch partitions peers.
  Prevention: one persisted room key + compatibility dual-subscribe window + warning telemetry.
- Failure mode: stale tunnel URLs remain selected.
  Prevention: stale classifier + TTL on endpoint candidates + auto demotion from selected transport.
- Failure mode: duplicate peer records from multiple discovery inputs.
  Prevention: canonical key merge using normalized prefixed address before storage.
- Failure mode: transport oscillation between relay and NKN.
  Prevention: add hysteresis/cooldown window before switching selected transport.

- Confirm endpoint list payload shows current cloudflare/nkn/local candidates and selected transport.
