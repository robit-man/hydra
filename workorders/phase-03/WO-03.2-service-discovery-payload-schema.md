## Implementation Checklist
1. Define normalized shape: `service`, `transport`, `base_url`, `local`, `tunnel`, `fallback`, `security`, `routes`.
2. Implement wrappers/proxies when upstream services cannot be modified.
3. Feed normalized payloads into router snapshot resolver.
4. Mark unknown/unsupported fields explicitly rather than omitting silently.

## Success Metrics
- Every enabled Hydra service appears in snapshot with normalized fields.
- Schema validator passes for all service payloads.
- Frontend can parse payloads without service-specific branching.

## Potential Failure Modes And Preventions
- Failure mode: inconsistent field names across services.
  Prevention: centralized schema + normalization utility.
- Failure mode: wrappers drift from upstream behavior.
  Prevention: include version/compat section in wrapper output.
- Failure mode: missing security route hints.
  Prevention: require `security` and `routes` blocks in schema.

## Verification
- `curl -s http://127.0.0.1:<router_port>/services/snapshot | jq '.snapshot.services'`
- Validate each service payload against schema.
