## Implementation Checklist
1. Add fallback blocks per service with state, base URLs, and endpoint variants.
2. Include `selected_transport` and selection reason.
3. Include stale tunnel URL and tunnel error details.
4. Ensure local/loopback endpoints are flagged non-routable for remote mode.

## Success Metrics
- `resolved.<service>.fallback` exists for every service.
- `selected_transport` always present and one of `cloudflare|upnp|nats|nkn|local`.
- Stale tunnel state preserved when active tunnel is gone.

## Potential Failure Modes And Preventions
- Failure mode: frontend picks invalid fallback endpoint.
  Prevention: add endpoint routability flags (`is_loopback`, `is_public`).
- Failure mode: ambiguous selected transport.
  Prevention: deterministic selection order and explicit rationale string.
- Failure mode: stale URL silently treated as healthy.
  Prevention: state machine with `stale` and `error` states.
