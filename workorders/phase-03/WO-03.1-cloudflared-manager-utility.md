## Implementation Checklist
1. Implement binary discovery and optional auto-install.
2. Launch cloudflared process and parse tunnel URL from stdout.
3. Add exponential restart backoff and rate-limit detection.
4. Track `active_url`, `stale_url`, `last_error`, `restarts`.

## Success Metrics
- Manager transitions through `starting -> active -> stale/error` states correctly.
- Stale URL is preserved when process exits after URL acquisition.
- Restart backoff increases on repeated failures and resets on success.

## Potential Failure Modes And Preventions
- Failure mode: infinite rapid restart loop.
  Prevention: bounded exponential backoff with cap.
- Failure mode: stale URL reused as active.
  Prevention: separate `tunnel_url` vs `stale_tunnel_url` fields.
- Failure mode: install attempts in restricted environments fail repeatedly.
  Prevention: explicit `auto_install_cloudflared` flag and failure cache.
