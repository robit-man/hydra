## Implementation Checklist
1. Add pending resolve map keyed by request ID with event + timeout.
2. Handle inbound `resolve_tunnels` requests and return `resolve_tunnels_result`.
3. Handle inbound `resolve_tunnels_result` and resolve pending promises.
4. Expose pending/timeout/success counters in telemetry.

## Success Metrics
- `POST /nkn/resolve` local mode returns immediate snapshot when target is self/empty.
- Remote mode returns timeout `504` on no reply and success payload on reply.
- Pending map drains to zero after timeout or completion.

## Potential Failure Modes And Preventions
- Failure mode: leaked pending entries.
  Prevention: TTL sweeper and explicit cleanup on all exit paths.
- Failure mode: request ID collisions.
  Prevention: UUID-like IDs with source prefix and monotonic suffix.
- Failure mode: replayed stale responses.
  Prevention: correlate by request ID and source address.
