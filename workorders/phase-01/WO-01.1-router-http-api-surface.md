## Implementation Checklist
1. Add endpoints: `GET /`, `GET /api`, `GET /health`, `GET /services/snapshot`, `GET /nkn/info`, `POST /nkn/resolve`.
2. Return stable JSON envelopes including `status`, `service`, `network`, and route catalog.
3. Include uptime, request counters, and NKN readiness fields.
4. Keep existing relay behavior backward compatible.

## Success Metrics
- All six endpoints return JSON with HTTP 200 in healthy state.
- `/health` includes uptime, request count, and NKN readiness.
- `/api` routes block matches implemented endpoints exactly.

## Potential Failure Modes And Preventions
- Failure mode: API thread blocks relay workers.
  Prevention: Run control-plane in separate thread/process loop.
- Failure mode: Route schema drift over time.
  Prevention: centralize route map in one payload builder.
- Failure mode: missing CORS/headers for browser access.
  Prevention: explicitly set JSON response headers and CORS policy.
