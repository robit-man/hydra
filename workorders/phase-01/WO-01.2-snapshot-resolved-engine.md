## Implementation Checklist
1. Poll each service with retry and timeout.
2. Accept source payloads from `/health`, `/router_info`, or `/tunnel_info` and normalize them.
3. Preserve last known-good snapshot when temporary polls fail.
4. Compute `resolved` with selected transport, local/tunnel/fallback endpoints.

## Success Metrics
- `/services/snapshot` contains `snapshot.services` and `snapshot.resolved` consistently.
- Each resolved service includes `transport`, `base_url`, and fallback metadata.
- Temporary probe failures do not erase previous good endpoints.

## Potential Failure Modes And Preventions
- Failure mode: Snapshot flaps between empty/non-empty.
  Prevention: last-known-good cache with timestamp and stale marker.
- Failure mode: inconsistent field names break frontend.
  Prevention: explicit normalization map and schema tests.
- Failure mode: loopback endpoint chosen for remote use.
  Prevention: mark loopback as local-only in resolver output.

## Verification
- `curl -s http://127.0.0.1:<router_port>/services/snapshot | jq '.snapshot.resolved'`
- Stop one service temporarily and confirm stale/last-good behavior.
