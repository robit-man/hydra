## Success Metrics
- `POST /api/interop/marketplace/catalog/ingest` returns `201` and persisted provider catalog payload.
- `GET /api/interop/marketplace/catalog/providers` returns catalog list + `diagnostics` object.
- Repeated ingest for same provider updates existing row (no duplicates).
- `/health` contains marketplace external catalog diagnostics and freshness gate signal.
- Unauthorized ingest attempts return `401`; contract incompatibility returns `422`.

## Potential Failure Modes And Preventions
- Failure mode: unauthorized catalog injection from arbitrary clients.
  Prevention: require `x-admin-key` or `HYDRA_MARKETPLACE_INGEST_TOKEN` token on ingest route.
- Failure mode: contract drift causes silent incompatible payload acceptance.
  Prevention: strict interop contract name + semver compatibility check before DB write.
- Failure mode: provider listing hides non-hydra sources unintentionally.
  Prevention: only apply source filter when explicitly provided; default is unfiltered listing.
- Failure mode: stale catalog not visible to operators.
  Prevention: expose freshness counters + last ingest timestamps via `/health.marketplace.external_catalogs`.
- Failure mode: query boolean parsing treats `activeOnly=false` as truthy.
  Prevention: explicit query boolean parser for `true/false/1/0/yes/no`.
