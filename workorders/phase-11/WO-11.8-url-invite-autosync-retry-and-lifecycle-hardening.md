# WO-11.8 URL Invite AutoSync Retry And Lifecycle Hardening

## Objective
Harden NoClip URL invite ingress so `autoSync` invites retry deterministically (bounded attempts + expiry), avoid stale timer bleed-over across invites, and reduce UI noise during background retry cycles.

  - pending invite lifecycle helpers/scheduler near `schedulePendingInviteSyncRetry` + `processPendingInviteSyncRetry` (`~1062-1179`).
  - bridge target apply flow return contract + silent mode in `applyNoClipBridgeTarget` (`~1181-1268`).
  - invite event ingestion and retry scheduling in `hydra-noclip-bridge-target` listener (`~1270-1320`).

## Implementation Checklist
1. Add bounded retry parameters (attempt cap, backoff base/max, max invite age).
2. Persist pending invite metadata (`syncAttempts`, timestamps, `lastError`, retry reason).
3. Add retry scheduler/processor that:
   - retries only for URL invites with `autoSync=true`,
   - aborts on expiry/attempt exhaustion,
   - clears pending state on success.
4. Update `applyNoClipBridgeTarget` to return structured sync result and support `silent` mode.
5. Update URL invite event handler to schedule retries only for non-terminal failures.
6. Reset/replace pending invite + timer when a new URL invite arrives.
7. Support `requestSync(..., { silent: true })` to suppress repetitive badge spam during retries.

## Success Metrics
- Auto-sync invite retries occur after initial sync failure and stop at configured cap.
- Pending invite state is cleared after successful sync or terminal failure.
- Opening a second invite URL cancels previous retry timer/state and tracks the new invite only.
- Background retry attempts do not repeatedly scroll/focus nodes or emit duplicate success badges.
- Syntax checks pass for touched JavaScript modules.

## Potential Failure Modes And Preventions
- Failure mode: infinite retry loop on unreachable peers.
  Prevention: enforce max attempts + invite age expiry before clearing pending state.
- Failure mode: stale timer from prior invite mutates a newer invite.
  Prevention: clear timer/state before storing a new URL invite payload.
- Failure mode: retry path floods operator badges/logs.
  Prevention: silent apply path + `requestSync` silent badge suppression.
- Failure mode: non-retryable parse/config errors continue retrying.
  Prevention: terminal error detection (`invalid target`, bridge-node creation failure, missing peer).
