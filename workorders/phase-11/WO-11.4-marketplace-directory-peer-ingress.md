# WO-11.4 Marketplace Directory Peer Ingress

## Objective
Convert NoClip marketplace provider-directory results into actionable Hydra peer entries so providers discovered through federation can be reached without manual copy/paste peer onboarding.

## Implementation Checklist
1. Add deterministic ingest method in PeerDiscovery for marketplace directory catalogs.
2. Parse router addresses from multiple provider fields (`router_nkn`, address arrays, source address).
3. Upsert peer entries with source metadata (`marketplace.directory`) and candidate transport info.
4. Prevent duplicate imports and avoid ping storms by default.
5. Surface import counts in marketplace directory UI text and refresh badge.
6. Document verification in runbook and rollout checklist.

## Success Metrics
- Directory refresh produces import summary (`imported/added/updated`) from catalog response.
- Imported peers appear in Hydra peer list without manual entry.
- Directory text includes peer count (`peers <m>`) after successful refresh.
- Ingest path is idempotent; repeated refresh updates existing peers instead of duplicating.

## Potential Failure Modes And Preventions
- Failure mode: malformed catalog rows crash peer ingest.
  Prevention: strict object/array guards and fallback parsing for every field.
- Failure mode: same peer imported repeatedly as duplicate entries.
  Prevention: dedupe by normalized pub key before upsert.
- Failure mode: refresh triggers mass network pings.
  Prevention: default `ping=false` with explicit bounded `pingLimit` option.
- Failure mode: UI shows provider counts but hides peer-ingress state.
  Prevention: append imported peer count in directory preview/badge text.
