# WO-11.1 Marketplace Catalog Federation Sync

## Objective
Add deterministic Hydra router catalog federation so provider/service state can be published repeatedly to external marketplace/interop endpoints with manual control and automatic retry/backoff.

## Implementation Checklist
1. Add typed config section for marketplace sync controls:
- auto publish toggle
- target URL list
- auth token/header settings
- publish timeout/interval/backoff.
2. Add router runtime sync state machine with:
- in-flight lock,
- last attempt/success/failure,
- recent result history,
- next due timestamp.
3. Add control-plane APIs:
- `GET /marketplace/sync`
- `POST /marketplace/catalog/publish`.
4. Add automatic publisher tick in status monitor with safe backoff.
5. Add CLI helper for operators to trigger manual publish and dry-runs.

## Success Metrics
- Router publishes catalog envelopes to one or more configured HTTP targets.
- Failed publishes produce explicit errors and exponential retry delay, not tight loops.
- Manual publish is available without waiting for auto interval.
- `/health` and dashboard payloads expose sync status for observability.

## Potential Failure Modes And Preventions
- Failure mode: rapid retry floods remote endpoint during outage.
  Prevention: bounded exponential backoff (`max_backoff_seconds`) and single in-flight guard.
- Failure mode: sync API triggers concurrent publish races.
  Prevention: explicit in-flight lock and `409` on concurrent manual publish attempts.
- Failure mode: auth token leaks through health or sync status payloads.
  Prevention: token presence is surfaced as boolean only; sensitive keys remain redacted.
- Failure mode: invalid target URL strings crash publisher.
  Prevention: URL normalization filters non-http(s) entries and skips invalid values safely.
