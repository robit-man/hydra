# WO-11.3 Hydra Frontend Federated Provider Directory

## Objective
Expose NoClip federated provider-directory state inside Hydra frontend so operators can see whether externally ingested marketplace catalogs are reachable and fresh without leaving the Hydra dashboard.

## Implementation Checklist
1. Add frontend config keys for NoClip API base URL, refresh interval, and last directory status timestamp.
2. Add UI row in marketplace summary card for directory counts and manual refresh button.
3. Poll `GET /api/interop/marketplace/catalog/providers` from frontend with:
- deterministic query defaults (`sourceNetwork=hydra`, `activeOnly=true`, `limit=100`),
- request timeout,
- stale-safe status updates.
4. Surface directory success and failure states in the card text.
5. Persist last known directory status for reload continuity.
6. Add operator runbook check for frontend directory visibility.

## Success Metrics
- Hydra UI shows `Directory: <count> providers • fresh <n> • stale <n>` after successful fetch.
- Manual refresh button executes fetch immediately and updates card text.
- API errors/timeouts render deterministic degraded text (`Directory: <reason>`) without breaking rest of panel.
- Directory polling repeats on configured interval and does not create concurrent in-flight storms.

## Potential Failure Modes And Preventions
- Failure mode: invalid API base string crashes directory polling.
  Prevention: normalize/fallback API base URL to `http://127.0.0.1:3001`.
- Failure mode: repeated refresh clicks spawn overlapping requests.
  Prevention: in-flight guard + temporary disable of refresh button.
- Failure mode: failed directory fetch resets catalog summary UI or crashes render path.
  Prevention: directory state updates are isolated from catalog/health rendering.
- Failure mode: long-hanging fetch blocks update loop.
  Prevention: hard timeout (`AbortController`) and deterministic error mapping.
