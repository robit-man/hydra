# WO-11.5 Peer Modal Bridge Target Orchestration

## Objective
Make the peer-modal `Bridge` action operational by wiring it to NoClip Bridge node orchestration so operators can select a discovered NoClip peer and immediately bind it to a usable `NoClipBridge` node without manual settings edits.

  - extend frontend sanity checks with peer-modal bridge targeting behavior around lines 53-57.

## Implementation Checklist
1. Update peer modal `Bridge` action to emit a structured event containing target pubkey/address/display name.
2. Add main-thread handler that:
- validates incoming target pubkey format,
- finds existing `NoClipBridge` nodes,
- chooses preferred node (selected node first, then unbound),
- creates a `NoClipBridge` node when none exist.
3. Apply selected target to the chosen node via `NoClipBridge.setTargetPeer`.
4. Refresh bridge dropdown/state and provide deterministic badge/log feedback.
5. Preserve failure visibility with explicit error badges on invalid target or node-create failure.
6. Update runbook and rollout docs with manual verification flow.

## Success Metrics
- Clicking `Bridge` on a NoClip peer configures an existing `NoClipBridge` node or creates one automatically.
- Target peer is persisted in the selected node config (`targetPub`/`targetAddr`) without opening node settings.
- Bridge node log shows target assignment and UI badge reports success.
- Invalid/empty peer identifiers fail fast with visible operator error and no partial mutation.

## Potential Failure Modes And Preventions
- Failure mode: bridge click does nothing because event path is disconnected.
  Prevention: use explicit `window` custom event (`hydra-noclip-bridge-target`) and handle dispatch failure.
- Failure mode: target applied to wrong bridge node in multi-node graphs.
  Prevention: deterministic node pick order (selected node, then unbound node, then first existing node).
- Failure mode: invalid target corrupts node config.
  Prevention: strict 64-hex pubkey validation before mutation.
- Failure mode: newly created node is not reflected in dropdown immediately.
  Prevention: immediate + deferred dropdown refresh (`setTimeout`) after assignment.
