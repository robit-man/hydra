# WO-11.7 Smart Invite Egress And UI Hook Alignment

## Objective
Align Smart Object invite egress with current NoClipBridge UI/runtime so invite QR generation is reachable from active bridge nodes and emits URLs compatible with the new Hydra invite-ingress parser contract.

## Implementation Checklist
1. Add explicit invite button in NoClipBridge node controls.
2. Dispatch a dedicated event (`hydra-noclip-smart-invite-open`) with node context from graph UI.
3. Refactor Smart Invite modal to open from event + node id and read current bridge config.
4. Emit invite URLs containing parser-compatible params (`noclip`, `bridgeNodeId`, `autoSync`, context fields).
5. Preserve backward-compatible metadata fields (`hydra`, `node`, `kind`) for existing scanners.
6. Remove brittle NodeStore monkey-patching and outdated DOM selectors.
7. Update runbook/rollout validation instructions.

## Success Metrics
- `Invite` control is visible on active NoClipBridge nodes and opens the Smart Invite modal.
- Generated URL includes bridge target pubkey and configured object/session context when present.
- QR generation works when QR library is available; URL-only fallback still works when unavailable.
- No runtime dependence on stale selectors (`data-node-id`, `.node-body`, `.node-content`) remains.

## Potential Failure Modes And Preventions
- Failure mode: invite button appears but does not open modal.
  Prevention: graph button dispatches explicit window event and modal registers listener on init.
- Failure mode: invite URL omits bridge context despite node config.
  Prevention: Smart Invite reads both direct config fields and `interopTarget`.
- Failure mode: invalid target pubkey creates unusable QR.
  Prevention: strict pubkey normalization + explicit badge error on missing/invalid target.
- Failure mode: removing monkey patch breaks invite access.
  Prevention: native graph control is the primary entrypoint and no hidden patch dependency remains.
