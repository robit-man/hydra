# WO-09.1 Rollout Gate Orchestrator And Artifact Unification

## Objective
Create one command that executes Hydra/NoClip interop rollout gates end-to-end (smoke, failure drills, router gates, optional backend gates) and emits one consolidated artifact for release decisions.

## Files To Touch (Hydra)
- `service_router/tools/hydra_noclip_rollout_gate.py` (new, line 1)
- `service_router/tools/hydra_noclip_interop_smoke.py` (imported by orchestrator, lines 526-570)
- `service_router/tools/hydra_noclip_failure_drill.py` (imported by orchestrator, lines 286-330)
- `docs/HYDRA_NOCLIP_INTEROP_ROLLOUT.md` command/runbook updates (lines 15-35, 50-75, 87-92)
- `workorders/README.md` phase index update (lines 43-46)
- `docs/HYDRA_TELEOP_UNIFICATION_WORKORDERS.md` phase listing + execution ordering (lines 71-74, 521-579)

## Implementation Checklist
1. Add `hydra_noclip_rollout_gate.py` that runs:
- interop smoke harness,
- failure drill harness,
- router `/health` rollout gate checks,
- router `/nkn/resolve` rollout gate checks,
- optional backend `/health` and `/api/da3/health` checks.
2. Emit a single JSON artifact with:
- per-check pass/fail,
- stage/drill rollups,
- timing,
- explicit `failed_checks` list,
- pointers to smoke/drill sub-artifacts.
3. Support deterministic simulation mode for CI and offline verification.
4. Add strict options for rollout safety:
- `--strict-live` passthrough to smoke stages,
- `--require-backend-health` when backend gate URLs are mandatory.
5. Update rollout runbook to make orchestrator command the primary gate command.

## Success Metrics
- One command can gate rollout and return non-zero on any required gate failure.
- JSON artifact contains enough detail to audit which gate failed and why.
- Simulated mode runs locally with no live services and still validates orchestration logic.
- Runbook contains an executable command sequence aligned with gate script flags.

## Potential Failure Modes And Preventions
- Failure mode: orchestration hides underlying smoke/drill artifacts.
  Prevention: include sub-artifact paths and embed key summaries in the consolidated artifact.
- Failure mode: missing backend URLs accidentally treated as success.
  Prevention: add `--require-backend-health` to force hard failure when backend checks are required.
- Failure mode: router health and resolve checks drift semantically.
  Prevention: evaluate both endpoints explicitly and fail if either rollout gate is not ready.
- Failure mode: local simulation diverges from live gate shape.
  Prevention: preserve identical result schema for simulated and live runs.
