## Implementation Checklist
1. Add inbound `service_rpc_request` handler in router.
2. Route request to selected service target/path with timeout and body-kind support.
3. Return `service_rpc_result` with status, headers, and body payload.
4. Add frontend `Net.nknServiceRpc(service, path, options)` helper.

## Success Metrics
- Browser can call a service path using only router NKN address.
- RPC helper supports JSON and binary/body_b64 responses.
- Timeouts and service errors map to explicit result status.

## Potential Failure Modes And Preventions
- Failure mode: oversized RPC payloads break NKN transport.
  Prevention: chunking or size limits with clear errors.
- Failure mode: path traversal/security risk.
  Prevention: strict allowlist for service + path patterns.
- Failure mode: dangling pending RPC promises.
  Prevention: per-request timeout cleanup.
