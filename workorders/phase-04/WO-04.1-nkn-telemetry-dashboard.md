## Implementation Checklist

2. Track per-peer usage and endpoint hit totals.
3. Add periodic sampler for throughput history.
4. Serve `/dashboard/data` with bounded history and peer list.

## Success Metrics
- `/dashboard/data` returns telemetry totals, history, and peer usage.
- Counter increments are observable during NKN traffic.
- Endpoint hit aggregation reflects service usage.

## Potential Failure Modes And Preventions
- Failure mode: unbounded telemetry memory growth.
  Prevention: capped deques and peer limits.
- Failure mode: lock contention affects request latency.
  Prevention: short critical sections and snapshot copies.
- Failure mode: missing counters on error paths.
  Prevention: increment counters in shared send/receive wrappers.
