## Success Metrics
- NoClip user can complete provider setup and publish at least one offer from frontend only.
- Offer lifecycle transitions update backend state and frontend badges in near real time.
- Router binding challenge + verification completes through UI.
- Friends/scene marketplace views reflect configured offer visibility.

## Potential Failure Modes And Preventions
- Failure mode: unauthenticated calls silently fail.
  Prevention: central API client auth guard + explicit re-auth prompts.
- Failure mode: invalid binding proof accepted.
  Prevention: backend verification remains authoritative; UI only submits signed proof.
- Failure mode: frontend and backend visibility enums drift.
  Prevention: shared enum mapping module and schema validation.
- Failure mode: stale local cache shows incorrect offer state.
  Prevention: refetch after state-changing calls and subscribe to transport updates.
