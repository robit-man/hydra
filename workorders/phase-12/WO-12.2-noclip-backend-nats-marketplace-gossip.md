## Implementation Checklist
1. Define env schema for `MARKETPLACE_NATS_ENABLED`, broker URLs, room/network subjects, publish and subscribe toggles.
2. Implement `MarketplaceGossipService` with:
- reconnect/backoff,
- subject subscriptions,
- checksum/time-window dedupe,
- publish helpers for catalog/status envelopes.
3. Wire service lifecycle into app startup/shutdown.
4. On REST or transport catalog ingest, publish normalized catalog to NATS subject.
5. On NATS receive, validate interop contract and ingest into same marketplace service path.
6. Expose gossip diagnostics in health/admin payload (connected, last publish, last receive, errors).

## Success Metrics
- NoClip backend subscribes to configured NATS subjects and ingests remote Hydra catalogs automatically.
- Locally updated catalogs are published to NATS and visible from another subscriber.
- Duplicate envelopes (same checksum/message id) are ignored without state churn.
- Health endpoint reports gossip connectivity and recent activity.

## Potential Failure Modes And Preventions
- Failure mode: message loops between ingest and republish.
  Prevention: source stamping + dedupe cache keyed by checksum/message id/source.
- Failure mode: broker outage stalls request path.
  Prevention: gossip is async/background; ingest path remains non-blocking.
- Failure mode: invalid contract envelopes pollute directory.
  Prevention: contract/version compatibility checks before ingest.
- Failure mode: unbounded dedupe memory growth.
  Prevention: fixed-size/TTL dedupe cache.
