# WO-12.5 Hydra NoClip Bridge Marketplace Broadcast

## Objective
Upgrade Hydra `NoClipBridge` catalog publishing from target-bound DM fanout to optional discovery-room broadcast so marketplace visibility works without manual target binding.

## Files To Touch
- `docs/js/noclipBridge.js`
  - discovery setup (`~1191-1240`).
  - inbound discovery DM processing (`~1325+`, `~1542+`).
  - catalog publish logic currently iterating `NODE_STATE` targets (`~2906-2988`).
- `docs/js/nodeStore.js`
  - add bridge defaults for marketplace broadcast mode and subject/channel config.
- `docs/js/main.js`
  - `hydra-provider-publish-catalog` event handling (`~213-230`) and UI feedback.
- `docs/index.html`
  - optional control for broadcast mode and publish scope.

## Implementation Checklist
1. Add bridge config options:
- publish scope (`targeted`, `broadcast`, `both`),
- discovery room/subject override,
- status publish toggle.
2. Implement discovery broadcast path for market catalog/status.
3. Keep targeted DM path for explicit peer sync and ack-required cases.
4. Add acknowledgement/error telemetry for broadcast publishes.
5. Update UI badges to clearly report broadcast vs targeted outcomes.

## Success Metrics
- Publishing catalog in broadcast mode reaches discovery participants without preconfigured target peers.
- Targeted mode continues to work for selected peers.
- Bridge logs and badges report attempted/sent/failed counts per publish scope.
- No regression to existing overlay/media bridge messaging.

## Potential Failure Modes And Preventions
- Failure mode: broadcast leaks private/friends-only offers.
  Prevention: enforce visibility filter before broadcast (`public` only unless explicit audience token).
- Failure mode: publish storm from frequent catalog updates.
  Prevention: debounce + checksum unchanged suppression.
- Failure mode: DM and broadcast paths create duplicate peer entries.
  Prevention: unify dedupe key (`providerFingerprint + checksum + ts`).
- Failure mode: bridge node without discovery connection silently drops publish.
  Prevention: explicit connection-state checks and surfaced operator errors.

## Verification
1. `node --check docs/js/noclipBridge.js docs/js/main.js docs/js/nodeStore.js`
2. With two clients in same discovery room, publish broadcast and verify remote receives catalog/status.
3. Switch to targeted mode and verify only selected peer receives update.
4. Validate no duplicate provider cards from receiving both targeted + broadcast copies.
