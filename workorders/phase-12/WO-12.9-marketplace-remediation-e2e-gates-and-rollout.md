# WO-12.9 Marketplace Remediation E2E Gates And Rollout

## Objective
Define and enforce end-to-end rollout gates for the marketplace remediation so NATS gossip discovery, provider configuration, and transport handlers ship safely.

## Files To Touch
- `service_router/tests/`
  - add router marketplace NATS/HTTP sync tests and stale-reject tests.
- `service_router/tools/`
  - add deterministic drill scripts for broker outage, duplicate events, and mixed transport publish.
- `../noclip-unified/noclip-backend/packages/api/src/services/marketplace.service.ts`
  - add unit/integration tests for ingest dedupe and quote/settle parity over transport vs REST.
- `../noclip-unified/noclip-backend/packages/api/src/transport/plugin.ts`
  - transport event contract tests.
- `docs/HYDRA_NOCLIP_MARKETPLACE_RUNBOOK.md`
  - operator procedures for NATS discovery and provider provisioning.
- `docs/HYDRA_NOCLIP_INTEROP_ROLLOUT.md`
  - go/no-go checklist and rollback procedure updates.

## Implementation Checklist
1. Add automated tests for each remediated gap:
- transport handlers live,
- NATS publish/subscribe ingest,
- provider UI config persistence,
- duplicate/stale event rejection.
2. Add chaos drills:
- broker unavailable,
- delayed/out-of-order catalogs,
- replayed quote/ticket messages.
3. Define rollout gates with explicit pass/fail thresholds.
4. Add rollback switches (disable NATS gossip, revert to HTTP directory mode, disable transport marketplace events).
5. Document operator runbook with command-level verification.

## Success Metrics
- Full cross-platform E2E test passes: Hydra provider publish -> NoClip discovery -> offer visible/usable.
- Gate checklist produces clear PASS/FAIL with artifacts.
- Rollback toggles can restore known-safe mode in under 5 minutes.
- No high-severity regression in existing relay/overlay functionality.

## Potential Failure Modes And Preventions
- Failure mode: remediation passes unit tests but fails in mixed network conditions.
  Prevention: multi-node E2E harness with induced latency/packet loss/replay.
- Failure mode: rollout proceeds with partial feature readiness.
  Prevention: hard gate requiring all critical test suites green.
- Failure mode: operator cannot quickly disable faulty gossip path.
  Prevention: documented runtime kill switch env/config flags.
- Failure mode: post-deploy drift between Hydra and NoClip contract versions.
  Prevention: interop contract compatibility test in CI and startup checks.

## Verification
1. Run full Hydra + NoClip test suites including new marketplace remediation tests.
2. Execute drill scripts and store logs/artifacts in rollout report.
3. Validate runbook procedures from clean environment.
4. Confirm rollback switches function during simulated incident.
