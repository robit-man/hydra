# WO-12.8 Cross-Platform Catalog Reconciliation And Health

## Objective
Unify catalog reconciliation, freshness checks, and health reporting across Hydra router, Hydra frontend, and NoClip backend/frontend.

## Files To Touch
- `service_router/router.py`
  - catalog generation and sync state payload (`~9409-9562`, `~9609+`, `~10163+`).
- `docs/js/main.js`
  - marketplace summary state handling (`~413-520`, `~801-819`).
- `docs/js/noclipBridge.js`
  - checksum/timestamp and stale rejection handling (`~2990-3044`).
- `../noclip-unified/noclip-backend/packages/api/src/app.ts`
  - health and rollout readiness payload (`~112-230`).

  - external catalog diagnostics and freshness logic.
- `../noclip-unified/earth/refactor/scene/PlanetScene.js`
  - marketplace summary status text and telemetry fields (`~1655-1753`).

## Implementation Checklist
1. Standardize reconciliation keys: `providerFingerprint`, `catalogChecksum`, `generatedAtMs`, `sourceNetwork`.
2. Apply one freshness policy across systems (max staleness window + tie-break rules).
3. Add health fields for gossip status, last catalog receive/publish, stale reject counts.
4. Normalize catalog/status envelope schema across transport, HTTP, and NATS paths.
5. Ensure UI badges map directly to backend/router health fields.

## Success Metrics
- All platforms compute same winner when two catalogs conflict.
- Health endpoints expose enough data to explain why a provider appears stale/offline.
- Hydra and NoClip UIs show consistent provider health and transport state.
- Stale catalog replay tests show deterministic rejection.

## Potential Failure Modes And Preventions
- Failure mode: clock skew causes valid catalogs to be rejected.
  Prevention: skew tolerance window + checksum fallback.
- Failure mode: differing checksum serialization causes false mismatches.
  Prevention: canonical JSON serialization helper shared across runtimes.
- Failure mode: health states appear inconsistent between layers.
  Prevention: derive UI status from standardized backend fields only.
- Failure mode: overly aggressive stale policy hides valid providers.
  Prevention: configurable TTL and diagnostics endpoint to inspect drops.

## Verification
1. Contract test: same input envelopes produce same checksum in Hydra and NoClip.
2. Replay test: older timestamp with same provider fingerprint is rejected.
3. `curl` health snapshots from router and backend show aligned status semantics.
4. UI smoke test: provider health badge transitions match backend events.
