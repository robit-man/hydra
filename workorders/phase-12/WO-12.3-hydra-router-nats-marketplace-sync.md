# WO-12.3 Hydra Router NATS Marketplace Sync

## Objective
Upgrade Hydra router marketplace sync from HTTP target push-only to dual-path sync with first-class NATS gossip publish/subscribe.

## Files To Touch
- `service_router/router.py`
  - marketplace sync config normalization (`~7163-7203`).
  - sync payload (`~9588-9607`) and publish loop (`~9669-9815`).
  - control-plane routes (`~10500-10613`).
  - fallback map currently carrying inactive NATS fields (`~7543-7547`).
- `service_router/router_config.json`
  - add `marketplace_nats` section and defaults near `marketplace_sync` (`~78-90`).
- `service_router/requirements-router.txt` or router bootstrap dependency list (if separate)
  - add NATS Python client dependency.
- `service_router/tools/hydra_catalog_publish.py`
  - add NATS publish/diagnostic options.

## Implementation Checklist
1. Add router config schema for NATS marketplace sync:
- enable flags,
- broker URL(s),
- subjects,
- client name,
- publish interval/timeouts.
2. Implement router-side NATS client worker:
- connect/reconnect,
- publish catalog/status envelopes,
- subscribe and ingest remote catalogs.
3. Preserve existing HTTP target publish as optional secondary path.
4. Extend `/marketplace/sync` payload to include NATS sync state (connected, last publish/receive, errors).
5. Add control-plane APIs for manual NATS publish + dry-run diagnostics.

## Success Metrics
- Router publishes catalog/status envelopes to NATS subject when enabled.
- Router ingests remote catalog envelopes from NATS and updates catalog cache.
- HTTP target sync remains functional and independently toggleable.
- `/marketplace/sync` exposes both HTTP and NATS sync state.

## Potential Failure Modes And Preventions
- Failure mode: sync worker blocks router main loop.
  Prevention: run NATS I/O on dedicated thread/task with queue boundary.
- Failure mode: accidental double publish storm (HTTP + NATS) at high frequency.
  Prevention: shared throttling and checksum-change gating.
- Failure mode: sensitive auth token leakage in sync status.
  Prevention: expose booleans/metadata only, never raw secret values.
- Failure mode: stale remote catalogs overwrite fresher local state.
  Prevention: generated timestamp and checksum freshness guards.
