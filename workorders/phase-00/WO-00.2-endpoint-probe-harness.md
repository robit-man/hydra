## Implementation Checklist
1. Build CLI to probe all configured services for `/health`, `/router_info`, `/tunnel_info`, and known model/list endpoints.
2. Emit JSON report with status code, latency, parse result, and schema-match verdict.
3. Return non-zero exit code on hard failures (timeout, invalid JSON where required, incompatible schema).
4. Add `--json`, `--strict`, and `--timeout` flags.

## Success Metrics
- `python3 service_router/tools/probe_endpoints.py --json` emits valid JSON.
- JSON output includes one result object per configured service.
- `--strict` mode fails when canonical keys are missing.

## Potential Failure Modes And Preventions
- Failure mode: Hanging probes block CI.
  Prevention: Per-request timeout and global timeout budget.
- Failure mode: False failures for optional endpoints.
  Prevention: Classify checks as required vs optional in schema file.
- Failure mode: Non-deterministic report ordering.
  Prevention: Sort service names before execution and output.
