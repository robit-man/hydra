## Validation Matrix

1. Visual conformance
   - Uniform thick border style in all views.
   - Left-to-right halftone progression visible and stable.
   - Black background with white/gray accent discipline.
2. Interaction conformance
   - Main navigation, subview navigation, overlay open/close, and vim aliases.
   - Scroll behaviors in debug and log dock.
3. Stability conformance
   - Repeated service state transitions do not corrupt frame.
   - Terminal resize stress test does not throw curses exceptions.
4. Accessibility/operability
   - Minimum size warning view for undersized terminals.
   - `--no-ui` mode remains functional with stdout logs.

## Suggested Test Procedures

- Start router and spam service state transitions:
  - toggle services on/off repeatedly in `Config` view.
- Induce watchdog health failures (temporarily stop ASR/TTS endpoints).
- Run terminal resize loop (wide -> narrow -> wide).
- Scroll deep debug history and log dock while updates stream in.

## Rollout Gates

1. Gate A: Symbol + border engine merged and stable.
2. Gate B: Layout + navigation migration complete.
3. Gate C: Log dock + print isolation complete.
4. Gate D: All views recomposed and keyboard parity verified.
5. Gate E: 30-minute soak without frame corruption.

## Failure Modes + Recovery

- If panel corruption appears:
  - disable halftone mode and verify structural border baseline first.
- If navigation regresses:
  - force keymap table dump in debug view and validate action routing.
- If throughput drops:
  - reduce log-dock refresh cadence and cap per-frame log render count.
