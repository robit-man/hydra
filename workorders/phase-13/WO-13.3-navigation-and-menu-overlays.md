## Implementation Plan

1. Replace `current_view` string branching with typed state:
   - `BASE_VIEW`
   - `OVERLAY_MENU`
   - `OVERLAY_HELP`
   - `OVERLAY_CONFIRM`
2. Add keymap tables per state, then route through one dispatcher.
3. Implement global navigation parity:
   - arrows + vim (`h/j/k/l`, `g/G`, PgUp/PgDn, Home/End).
4. Add overlay menu renderer and stack (open/close/switch) similar to btop's menu lifecycle.
5. Add contextual footer hints generated from active state/keymap.
6. Introduce command IDs for actions (instead of direct side effects inside `_handle_input`).

## Success Metrics

- Every key action is discoverable through a keymap table.
- Overlay menu can open from any view and return cleanly to prior state.
- Vim navigation works everywhere the arrow keys work.
- No direct view mutation outside the dispatcher.

## Failure Modes + Preventive Controls

- Failure: Key conflicts between debug scrolling and global navigation.
  - Prevention: priority-ordered key dispatch by active state.
- Failure: Overlay leaks state and traps user.
  - Prevention: guaranteed escape route (`ESC`) and stack pop invariant.
- Failure: Hard-coded strings drift from actual routes.
  - Prevention: define enum/constants for all states and actions.
