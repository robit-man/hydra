## Implementation Plan

1. Add a thread-safe UI log buffer to `EnhancedUI` with:
   - timestamp
   - source
   - level
   - message
2. Add bottom dock renderer:
   - dedicated fixed-height panel at screen bottom
   - own border/title
   - vertical scroll (`PgUp/PgDn`, optional `[ ]` fine step)
3. Add `append_runtime_log(...)` API on `EnhancedUI`.
4. Add `log_sink` callback support in `ServiceWatchdog` and replace all runtime `print(...)` calls with sink emission.
5. Wire router init to pass UI sink into watchdog.
6. Keep stdout fallback only when UI is disabled (`--no-ui` path).

## Success Metrics

- Watchdog state-transition lines never write directly into curses body.
- All watchdog/router runtime messages appear in bottom dock.
- Bottom dock is scrollable and retains backlog.
- UI remains visually stable under rapid service state churn.

## Failure Modes + Preventive Controls

- Failure: Multithreaded append races cause intermittent crashes.
  - Prevention: lock around buffer writes and snapshot copies for render.
- Failure: Log flood starves frame rate.
  - Prevention: bounded ring buffer and render-time clipping.
- Failure: Sink exceptions break watchdog loops.
  - Prevention: sink call wrapped in `try/except` and silent fallback.
