## Implementation Plan

1. Add a `compute_layout(width, height)` function returning named panel rects:
   - `hydra_panel`
   - `content_panel`
   - `log_dock`
   - `status_strip`
2. Move all manual width/height calculations out of `_main`.
3. Enforce min-size rules and explicit degrade mode:
   - If too small, render a dedicated "terminal too small" panel.
4. Add panel title placement policy:
   - Top title in border notch style (btop `title_left/title_right` concept).
5. Ensure all render functions receive panel-local dimensions only.
6. Add optional panel weight config for future user customization.

## Success Metrics

- Single source of truth for all panel coordinates.
- No panel overlaps at any terminal size >= minimum.
- Resize events preserve stable panel boundaries and do not smear previous frames.
- Every view draws inside the panel rect returned by the layout engine.

## Failure Modes + Preventive Controls

- Failure: Off-by-one writes at right edge trigger curses errors.
  - Prevention: enforce width budget helper and clamp max x per panel.
- Failure: Tiny terminals cause negative widths/heights.
  - Prevention: hard min clamp and explicit size-error state.
- Failure: View renderers still assume full-screen coordinates.
  - Prevention: pass panel-local origin + dimensions and remove direct global references.
