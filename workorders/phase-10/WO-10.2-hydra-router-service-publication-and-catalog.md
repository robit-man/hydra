## Implementation Checklist
1. Add router-level catalog endpoint (for example `GET /marketplace/catalog`) that includes:
- provider identity (`router_nkn`, provider key fingerprint)
- service list with health/status/capacity
- endpoint candidates (`nkn`, `cloudflare`, `local`, optional `nats`)
- current selected transport and stale reasons.
2. Add subordinate repository service publication fields in `router_config.json` and runtime override support.
3. Reuse existing snapshot/resolved logic so catalog and `/services/snapshot` stay consistent.
4. Include cloudflared error-state and stale URL metadata in catalog entries.
5. Add probe CLI to validate catalog completeness and endpoint reachability.

## Success Metrics
- Router exposes catalog data for all enabled services with parent + subordinate provenance.
- Catalog includes at least one reachable endpoint candidate per healthy service.
- Cloudflared stale/error states are visible and do not hide local/NKN fallbacks.
- Probe CLI returns pass/fail summary suitable for release gates.

## Potential Failure Modes And Preventions
- Failure mode: catalog and `/services/snapshot` disagree.
  Prevention: derive catalog from the same normalized snapshot object.
- Failure mode: stale cloudflared URL remains marked active.
  Prevention: strict stale detection and selected transport fallback to NKN/local.
- Failure mode: subordinate service metadata never refreshes.
  Prevention: periodic snapshot invalidation and forced refresh path in probe tool.
- Failure mode: provider pricing metadata leaks secrets.
  Prevention: publish only public quote fields; keep signing keys/private fields internal.

## Verification
- `curl` router catalog endpoint and verify per-service endpoint candidates and status.
- Run `hydra_catalog_probe.py` against local router.
- Toggle one service unhealthy and confirm catalog reflects degraded state quickly.
