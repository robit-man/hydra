## Implementation Checklist
1. Define billable units by service type (tokens, seconds, frames, MB, operations).
2. Add quote endpoint returning estimated and max charge before execution.
3. Record usage events with correlation ids linking request -> quote -> settlement.
4. Support provider-defined rate cards and optional consumer caps.
5. Add settlement journal entries and reconciliation reports.

## Success Metrics
- Every billable request has one quote record and one settlement record.
- Charges match actual metered usage within tolerance rules.
- Provider dashboards expose per-service revenue and volume.
- Consumer dashboards expose spend history and remaining credits.

## Potential Failure Modes And Preventions
- Failure mode: missing usage records for successful requests.
  Prevention: write-ahead usage event with completion/failure transition states.
- Failure mode: quote and settlement use different rate cards.
  Prevention: immutable quote snapshot referenced by settlement.
- Failure mode: transport path ambiguity breaks attribution.
  Prevention: enforce source transport tagging in all request envelopes.
- Failure mode: retries create duplicate settlement debits.
  Prevention: idempotency keys per charge attempt and unique DB constraints.
