## Implementation Checklist
1. Define shared visual tokens for transport/health/availability states across both apps.
2. Show provider-hosted service badges with online/degraded/offline and selected transport.
3. Add friend/public visibility indicator per service offer.
4. Add credit balance and estimated cost preview in both Hydra and NoClip UIs.
5. Keep manual peer add, QR ingress, and chat/bridge actions visually and behaviorally aligned.

## Success Metrics
- Users see the same status vocabulary and badge semantics across both frontends.
- Hosted services show live health and transport state with no stale UI drift.
- Credit and quote previews are visible before service invocation.
- Friend/public scope of offers is obvious and consistent.

## Potential Failure Modes And Preventions
- Failure mode: UI reflects stale backend/router state.
  Prevention: polling + event-driven refresh with timestamp freshness indicators.
- Failure mode: different status color meanings across apps.
  Prevention: shared status token map and component-level tests.
- Failure mode: quote preview hidden for some invocation paths.
  Prevention: central preflight UI hook before any service dispatch.
- Failure mode: manual peer UX diverges again over time.
  Prevention: shared checklist and visual regression snapshots for both frontends.
