## Implementation Checklist

- `market.access.ticket.issue/verify`

2. Add explicit `contract_version` and `compat_min_version` to every envelope and health payload.
3. Publish a normative JSON schema document and include example payloads for Hydra router, NoClip backend, and both frontends.
4. Add strict parser/normalizer helpers in Hydra and NoClip frontend so unknown fields are tolerated but missing required fields fail fast.
5. Add contract drift checks in health endpoints and rollout gates.

## Success Metrics
- Both `GET /health` surfaces (Hydra and NoClip backend) report the same contract version.
- Hydra resolve payloads and NoClip interop payloads validate against one schema bundle.
- Unknown/old payloads are rejected with deterministic error codes and diagnostics.
- Contract test fixtures pass in both repositories.

## Potential Failure Modes And Preventions
- Failure mode: envelope fields diverge between HTTP and NKN/NATS channels.
  Prevention: one source schema with transport-agnostic envelope fields and CI validation.
- Failure mode: silent compatibility break for older clients.
  Prevention: enforce `compat_min_version` checks and explicit downgrade errors.
- Failure mode: health endpoints claim ready while contract mismatches remain.
  Prevention: include contract compatibility in rollout gate status computation.
- Failure mode: frontend applies partially valid payloads.
  Prevention: centralized normalizers with required-field guards before state mutation.
