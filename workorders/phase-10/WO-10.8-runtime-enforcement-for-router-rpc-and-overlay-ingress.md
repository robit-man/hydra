## Implementation Checklist
1. Add router-side pre-execution enforcement hook for all billable service RPC calls.
2. Require valid quote/ticket/credit reservation prior to execution start.
3. Require overlay mutation permission checks for item/layer/overlay targets.
4. Return structured denial responses with machine-readable reason codes.
5. Ensure denied requests never emit side effects (no partial writes/streams).

## Success Metrics
- Unauthorized or underfunded requests are blocked before execution.
- Overlay mutation requests respect owner/viewer/editor policies consistently.
- Denials are visible in telemetry and user-facing status surfaces.
- Successful requests show matching enforcement audit trail.

## Potential Failure Modes And Preventions
- Failure mode: checks run after partial execution starts.
  Prevention: hard preflight gate before dispatch to service worker/process.
- Failure mode: overlay write path bypasses policy checks.
  Prevention: centralize mutation entrypoint and remove direct unsafe paths.
- Failure mode: denial reasons are opaque to users.
  Prevention: normalized error codes plus concise actionable messages.
- Failure mode: mixed success in multi-part payloads.
  Prevention: transactional envelope semantics with all-or-reject policy.

## Verification
- Attempt RPC without credit/ticket and verify preflight rejection.
- Attempt overlay mutation without write permission and verify denial + no mutation.
- Confirm successful authorized request executes and logs enforcement pass.
