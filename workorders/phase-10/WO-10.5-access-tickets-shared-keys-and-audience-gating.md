## Implementation Checklist
1. Introduce short-lived access tickets bound to provider, service id, consumer id, and scope (`infer`, `overlay.write`, `stream`).
2. Add signature verification and nonce/replay windows for ticket use in Hydra router.
3. Enforce audience policies (friends/public/private) before issuing tickets.
4. Add key rotation path with overlap window for old/new key ids.
5. Add error taxonomy for access denials (`ticket_expired`, `scope_denied`, `audience_denied`, `signature_invalid`).

## Success Metrics
- Service RPC and overlay mutation requests require valid, non-expired tickets.
- Replay attempts with reused nonce/ticket ids are rejected.
- Friend/public policy is enforced consistently in backend and router.
- Key rotation can occur without breaking active valid sessions.

## Potential Failure Modes And Preventions
- Failure mode: ticket replay enables unauthorized repeated inference calls.
  Prevention: nonce + jti replay cache with strict TTL.
- Failure mode: audience policy checked in one layer only.
  Prevention: enforce audience at ticket issue time and execution time.
- Failure mode: key rotation invalidates all in-flight sessions.
  Prevention: dual-key validation window and explicit `kid` usage.
- Failure mode: sensitive ticket fields leak into logs.
  Prevention: redact token payloads and log only fingerprints/claim ids.

## Verification
- Issue ticket, execute service RPC, and verify success.
- Retry with expired and replayed ticket and verify deterministic rejection.
- Rotate key config and confirm overlapping validation behavior.
