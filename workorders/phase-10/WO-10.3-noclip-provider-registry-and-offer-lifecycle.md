## Implementation Checklist
1. Add tables for provider profiles, router bindings, service offers, and offer visibility (`friends`/`public`).
2. Require verified wallet and authenticated account ownership before offer publication.
3. Support offer lifecycle states: `draft`, `published`, `paused`, `retired`.
4. Add API endpoints for CRUD + publish/unpublish + health-sync of offers.
5. Persist selected endpoint strategy and per-offer rate card metadata.

## Success Metrics
- Authenticated providers can create/edit/publish offers tied to registered Hydra endpoints.
- Offer visibility rules are enforced for friend-only vs public listings.
- Offer status automatically reflects router health sync (online/degraded/offline).
- Offer records are queryable by NoClip frontend without exposing private keys/seeds.

## Potential Failure Modes And Preventions
- Failure mode: user publishes offers for unowned router endpoints.
  Prevention: require signed endpoint ownership proof before provider binding.
- Failure mode: stale offers remain purchasable while service is offline.
  Prevention: health-synced availability flags and automatic pause on repeated health failure.
- Failure mode: friend-only offers leak into public listings.
  Prevention: audience filters enforced in query layer and transport fanout layer.
- Failure mode: wallet link churn invalidates provider identity unexpectedly.
  Prevention: immutable provider id + re-verification flow with clear state transitions.
