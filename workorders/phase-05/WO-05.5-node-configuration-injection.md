## Implementation Checklist
1. Add per-node fields: `service`, `endpointSource`, `endpointMode`, `lastResolvedAt`, `resolveDiagnostics`.
2. Auto-apply resolver output when `endpointSource=router`.

## Success Metrics
- Existing saved flows load without config errors.
- Nodes can switch between manual and router-resolved endpoint modes.
- Resolver updates do not overwrite manual mode values.

## Potential Failure Modes And Preventions
- Failure mode: breaking old flow JSON due missing keys.
  Prevention: non-breaking defaults in `NodeStore.ensure` path.
- Failure mode: resolver update loops on every render.
  Prevention: apply only on changed endpoint hash/timestamp.
- Failure mode: mixed mode confusion per node.
  Prevention: explicit `endpointSource` + status indicator in node UI.
