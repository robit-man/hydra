# WO-02.3 Single-Instance Lock + Desired State Persistence

## Objective
Prevent duplicate watchdog instances and persist service enabled-state across restarts.

## Reference Anchors (Teleoperation)

## Implementation Checklist
1. Add non-blocking lock file acquisition on startup.
2. Emit clear error when another watchdog instance is active.
3. Persist desired service enabled flags atomically.
4. Restore desired state on next boot.

## Success Metrics
- Second watchdog start exits immediately with lock-holder diagnostic.
- Desired on/off state survives restart.
- Corrupt state file is handled with warning and safe fallback.

## Potential Failure Modes And Preventions
- Failure mode: stale lock file blocks startup.
  Prevention: verify PID liveness before treating lock as active.
- Failure mode: torn writes corrupt desired state.
  Prevention: write temp file then atomic rename.
- Failure mode: lock not released on shutdown path.
  Prevention: release lock in `finally` and signal handlers.

## Verification
- Start watchdog twice and verify second process exits with lock error.
- Toggle service desired state, restart watchdog, verify state restored.
